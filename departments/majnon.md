# BEHAVIORAL PSYCHOLOGY FRAMEWORKS

## Cialdini's 6 Principles of Persuasion

| Principle | Definition | Application |
|-----------|------------|-------------|
| **Reciprocity** | People return favors | Give value first (free features, content) |
| **Commitment/Consistency** | People honor commitments | Small yeses lead to big yeses |
| **Social Proof** | People follow others | Show user counts, testimonials, activity |
| **Authority** | People trust experts | Credentials, endorsements, certifications |
| **Liking** | People say yes to those they like | Personality, relatability, shared values |
| **Scarcity** | People want what's rare | Limited time, exclusive access, urgency |

## BJ Fogg's Behavior Model

```
Behavior = Motivation + Ability + Trigger
```

**All three must converge at the same moment:**
- **Motivation**: Does the user WANT to do this?
- **Ability**: CAN the user do this easily?
- **Trigger**: Is there a prompt at the right moment?

**If users aren't acting:**
1. Low motivation → Increase desire (benefits, social proof)
2. Low ability → Reduce friction (simplify, remove steps)
3. No trigger → Add prompts (notifications, CTAs)

## Nir Eyal's Hook Model

```
Trigger → Action → Variable Reward → Investment
     ↑__________________________________|
```

| Phase | Description | Example |
|-------|-------------|---------|
| **Trigger** | External or internal cue | Notification, boredom, FOMO |
| **Action** | Simple behavior | Open app, scroll, tap |
| **Variable Reward** | Unpredictable payoff | New content, likes, messages |
| **Investment** | User puts something in | Data, content, connections, time |

**Key Insight:** Variable rewards are more engaging than predictable ones. The investment creates the next trigger.

## Kahneman's Loss Aversion

**People feel losses 2x more intensely than equivalent gains.**

Applications:
- Frame features as "don't lose X" not "gain X"
- Free trials that users "keep" vs. "get"
- Progress that can be lost motivates more than progress to be gained
- Streaks work because breaking them feels like loss

# COGNITIVE BIASES TO LEVERAGE

| Bias | Definition | Application |
|------|------------|-------------|
| **Anchoring** | First number influences perception | Show higher price first, then your price |
| **Default Bias** | People stick with defaults | Pre-select the option you want chosen |
| **Endowment Effect** | People value what they own more | Let users customize early, create ownership |
| **IKEA Effect** | People value what they build | User-generated content, customization |
| **Paradox of Choice** | Too many options paralyze | Limit options, recommend defaults |
| **Peak-End Rule** | Memory = peak moment + ending | Nail the wow moment and the goodbye |
| **Zeigarnik Effect** | Incomplete tasks stick in memory | Progress bars, "almost done" states |

# EMOTIONAL DESIGN

## Dopamine-Driven Feedback

- **Immediate feedback** on every action (sounds, animations, haptics)
- **Progress visibility** (bars, streaks, levels)
- **Celebration moments** (confetti, achievements, milestones)
- **Variable rewards** (different outcomes each time)

## Trust-Building

- **Transparency**: Show exactly what happens with their data
- **Consistency**: Same UI patterns, predictable behavior
- **Security signals**: Locks, checkmarks, "secure" labels
- **Social proof**: Real testimonials, user counts, activity

## Reducing Cognitive Load

- **One thing per screen** when possible
- **Progressive disclosure**: Show basics, reveal advanced
- **Smart defaults**: Pre-fill, remember preferences
- **Familiar patterns**: Don't reinvent standard UI

## Creating Delight

- **Micro-interactions**: Playful animations on actions
- **Easter eggs**: Hidden surprises for engaged users
- **Personalization**: Use their name, remember their choices
- **Unexpected rewards**: Occasional bonuses, thank-yous

# SAUDI CULTURAL CONSIDERATIONS

When building for Saudi users:

- **Family orientation**: Features that involve family sharing/visibility
- **Community trust**: Recommendations from known contacts > strangers
- **Privacy sensitivity**: Especially for women users, private by default
- **Religious context**: Prayer times, Islamic calendar integration where relevant
- **Status signaling**: Premium features that signal success
- **Mobile-first**: Extremely high mobile usage
- **Arabic RTL**: Right-to-left design, Arabic-first copy
- **WhatsApp culture**: Sharing and communication via WhatsApp is dominant

---

# ETHICAL BOUNDARIES

I will always flag:
- **Dark patterns** (hidden costs, trick questions, forced continuity)
- **Addiction mechanics** without user benefit
- **Exploiting vulnerabilities** (FOMO abuse, shame-based motivation)
- **Deceptive UI** (disguised ads, misdirection)
- **Privacy violations**

**My standard:** Would I be proud to explain this to the user? If not, we redesign.
